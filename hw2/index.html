Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau’s algorithm evaluates a Bezier curve at a parameter t by repeatedly linearly interpolating between adjacent control points. Starting from the original control points p0…pn, one subdivision step produces a new list of points p'i = (1−t)·pi + t·p{i+1}. Repeating this process reduces the list size by 1 each level until only one point remain and that final point is the point on the Bezier curve at parameter t.

In my implementation, BezierCurve::evaluateStep(points) performs exactly one level of this interpolation where it loops over each adjacent pair (points[i], points[i+1]) and pushes (1−t)·points[i] + t·points[i+1] into an output vector (size n−1).

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau’s algorithm extends to Bezier surfaces by treating the Bezier surface as two nested 1D Bezier evaluations (the surface is separable in u and v). Given a 2D grid of control points P(i,j), we first run 1D de Casteljau along one parameter direction to collapse the grid into a single set of intermediate control points, then run 1D de Casteljau again along the other parameter to obtain the final point on the surface at (u,v). (Evaluating u first then v, or v first then u, produces the same surface point.)

In my implementation, BezierPatch::evaluateStep(points, t) performs one de Casteljau step in 3D by linearly interpolating adjacent points: p'i = (1−t)·pi + t·p{i+1}. Then BezierPatch::evaluate1D(points, t) repeatedly calls evaluateStep until only one point remains.

To evaluate the full surface, BezierPatch::evaluate(u, v) applies this 1D routine twice: it evaluates u on each of the 4 Bezier curves in that direction to produce 4 intermediate points, then evaluates v on those 4 points to return the final surface point.
